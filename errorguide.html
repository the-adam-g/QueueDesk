QueueDesk Error Guide

Error #a1

Error #a1 occurs due to Unauthorised Table Access when attempting to export data to a CSV file.

Unauthorised Tables are defined in the constant BANNEDTABLES in config.php and can be changed by the server owner to fix the issue.

By default, to abide legislation such as GDPR and DPA, the table users which stores all user information such as their passwords is banned but can be changed.


Error #b1

Error #b1 occurs due to a CSRF violation.

CSRF stands for Cross Site Request Forgery and is a type of attack that uses forged post requests using a malicious redirect to abuse login credentials to steal data.

To protect against CSRF, QueueDesk uses the approved method of generation unique tokens upon login and form use which ensures the forms can only run if submitted through QueueDesk.

If #b1 occurs, it is because your CSRF token does not match the one generated by the form. To fix you should refresh the page or logout and re-login.


Error #b2

Error #b2 occurs due to a Forbidden Action.

In QueueDesk, a Forbidden Action refers to attempting to perform an operation without the permissions required or one banned by your administrator..
